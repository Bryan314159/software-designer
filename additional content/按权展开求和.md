### 一、先搞懂 2 个核心基础概念（听不懂这两个，公式就是天书）

按权展开的本质，就是搞懂「数字在不同位置，代表的实际大小不一样」，核心就是两个概念：**基数**、**数位**、**位权**（这里补充基数，是小白理解的前提）。

1. **基数**：一个进制里，能使用的不同数字的总个数，也叫进制数，用 R 表示。
    
    - 咱们天天用的十进制，能用 0-9 共 10 个数字，所以基数 R=10；
    - 二进制只有 0、1 两个数字，基数 R=2；
    - 八进制有 0-7 共 8 个数字，基数 R=8；
    - 十六进制有 0-9、A-F（A=10、B=11…F=15）共 16 个数字，基数 R=16。
    
2. **数位**：数字在一个数里的位置编号，**这是小白最容易踩坑的地方，必须记死规则**：
    
    - 整数部分：从 ** 小数点左边第一位（最靠右的整数位）** 开始，**从右往左**数，编号依次是 `0、1、2、3……`
    - 小数部分：从 ** 小数点右边第一位（最靠左的小数位）** 开始，**从左往右**数，编号依次是 `-1、-2、-3……`
        
        > 举个例子：十进制数 `123.45`
        > 
        > 整数部分数位：1（第 2 位）、2（第 1 位）、3（第 0 位）
        > 
        > 小数部分数位：4（第 - 1 位）、5（第 - 2 位）
        
    
3. **位权**：某个数位上的数字，代表的实际权重，大白话就是「这个位置上的数字 1，能代表多大的实际数值」。
    
    - 位权的计算公式：**基数 R 的「数位编号」次方**
    - 比如十进制（R=10）：第 0 位（个位）的位权是 10⁰=1，第 1 位（十位）是 10¹=10，第 2 位（百位）是 10²=100，第 - 1 位（十分位）是 10⁻¹=0.1，以此类推。

### 二、从你最熟悉的十进制，理解按权展开求和

其实咱们天天写的十进制数，本质就是按权展开求和的简写，先拿十进制入门，你绝对不会懵。

#### 例子：十进制数 `123.45` 的按权展开求和

1. 定基数：十进制，R=10
2. 标数位：1 (第 2 位)、2 (第 1 位)、3 (第 0 位) . 4 (第 - 1 位)、5 (第 - 2 位)
3. 按权展开：每一位数字 × 对应位权，用加号连接
    
    `1×10² + 2×10¹ + 3×10⁰ + 4×10⁻¹ + 5×10⁻²`
4. 求和计算：
    
    `1×100 + 2×10 + 3×1 + 4×0.1 + 5×0.01 = 100+20+3+0.4+0.05 = 123.45`

你看，咱们平时写的 123.45，就是把上面的展开式简化后的写法，按权展开只是把它还原成了最本质的计算逻辑。

### 三、核心通用规则：任意 R 进制转十进制（软考核心用途）

按权展开求和的唯一核心用途，就是**把任意进制的数，转换成咱们能看懂的十进制数**，通用规则一句话就能说清：

> 把 R 进制数的每一位数字，乘以它对应的位权（R 的数位编号次方），再把所有乘积加起来，最终的和就是对应的十进制数。

接下来咱们就讲软考最高频的 3 种进制的转换，每一个都给你举例子，步骤无跳跃，你跟着就能算。

### 四、软考高频场景实战（二进制 / 十六进制是重中之重）

#### 场景 1：二进制转十进制（软考每年必考，优先级最高）

二进制基数 R=2，只有 0 和 1 两个数字，位权是 2 的数位编号次方。

> 软考做题技巧：0 乘以任何数都是 0，所以数位上是 0 的，可以直接跳过不计算，大幅提速。

##### 例子 1：二进制整数 `1011` 转十进制

1. 标数位（从右往左，整数部分）：1 (第 3 位)、0 (第 2 位)、1 (第 1 位)、1 (第 0 位)
2. 按权展开：`1×2³ + 0×2² + 1×2¹ + 1×2⁰`
3. 求和计算：`1×8 + 0×4 + 1×2 + 1×1 = 8+0+2+1 = 11`

##### 例子 2：二进制带小数 `101.101` 转十进制

1. 标数位：整数 1 (第 2 位)、0 (第 1 位)、1 (第 0 位) . 小数 1 (第 - 1 位)、0 (第 - 2 位)、1 (第 - 3 位)
2. 按权展开：`1×2² + 0×2¹ + 1×2⁰ + 1×2⁻¹ + 0×2⁻² + 1×2⁻³`
3. 求和计算：`1×4 + 0×2 + 1×1 + 1×0.5 + 0×0.25 + 1×0.125 = 4+0+1+0.5+0+0.125 = 5.625`

> 小白必记：负次方的计算规则 `R⁻ⁿ = 1/(Rⁿ)`，比如 2⁻¹=1/2=0.5，2⁻²=1/4=0.25，2⁻³=1/8=0.125
> 
> 软考提速必背：2⁰到 2¹⁰的结果，背下来做题秒出答案：
> 
> 2⁰=1、2¹=2、2²=4、2³=8、2⁴=16、2⁵=32、2⁶=64、2⁷=128、2⁸=256、2⁹=512、2¹⁰=1024

#### 场景 2：十六进制转十进制（软考高频考点）

十六进制基数 R=16，数字规则：0-9 对应 0-9，**A=10、B=11、C=12、D=13、E=14、F=15**，这个对应关系必须记死，是小白最容易错的地方。

##### 例子 1：十六进制整数 `2AF` 转十进制

1. 先把字母转成对应数字：2=2、A=10、F=15
2. 标数位：2 (第 2 位)、A (10，第 1 位)、F (15，第 0 位)
3. 按权展开：`2×16² + 10×16¹ + 15×16⁰`
4. 求和计算：`2×256 + 10×16 + 15×1 = 512+160+15 = 687`

##### 例子 2：十六进制带小数 `2A.8` 转十进制

1. 字母转数字：2=2、A=10、8=8
2. 标数位：2 (第 1 位)、A (10，第 0 位) . 8 (第 - 1 位)
3. 按权展开：`2×16¹ + 10×16⁰ + 8×16⁻¹`
4. 求和计算：`2×16 + 10×1 + 8×0.0625 = 32+10+0.5 = 42.5`

#### 场景 3：八进制转十进制（软考低频考点，了解即可）

八进制基数 R=8，数字 0-7，位权是 8 的数位编号次方，方法完全一致。

例子：八进制 `127` 转十进制

展开求和：`1×8² + 2×8¹ +7×8⁰ = 64+16+7 = 87`

### 五、软考小白必避的 4 个高频易错点（90% 的新手都在这里丢分）

1. **数位编号数反 / 数错**
    
    整数部分必须**从右往左、从 0 开始**编号，很多新手从左往右数，或者从 1 开始数，直接全错。
    
    小技巧：先找小数点，贴着小数点左边的整数位，永远是第 0 位；贴着小数点右边的小数位，永远是第 - 1 位。
    
2. **十六进制字母对应错误**
    
    必须记死：A=10，F=15，绝对不能把 A 当成 11、F 当成 16，十六进制单个数字最大是 15（F），不可能出现 16。
    
3. **负次方计算错误**
    
    牢记 `R⁻ⁿ = 1/(Rⁿ)`，比如 8⁻¹=1/8=0.125，16⁻¹=1/16=0.0625，绝对不能当成 0.8、0.16。
    
4. **基数和位权搞混**
    
    几进制，位权就是几的次方，二进制用 2 的次方，十六进制用 16 的次方，不能用混。

### 六、软考真题实战（带你走一遍考试流程）

【软考历年真题示例】二进制数 110101 对应的十进制数是（）

A. 49 B. 53 C. 65 D. 67

解题步骤：

1. 标数位（从右往左）：1 (第 5 位)、1 (第 4 位)、0 (第 3 位)、1 (第 2 位)、0 (第 1 位)、1 (第 0 位)
2. 按权展开：`1×2⁵ + 1×2⁴ + 0×2³ + 1×2² + 0×2¹ + 1×2⁰`
3. 求和：`32 + 16 + 0 + 4 + 0 + 1 = 53`
4. 答案选 B，妥妥的送分题。

### 最后给你总结一个通用步骤口诀，方便记忆

1. **定基数**：几进制，基数 R 就是几
2. **标数位**：整数右往左从 0 起，小数左往右从 - 1 起
3. **乘位权**：每位数字 × R 的数位编号次方
4. **加总和**：所有乘积相加，就是最终的十进制数
